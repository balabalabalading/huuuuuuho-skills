# mp-article-writor

一个 Claude Code Skill，帮助自媒体创作者将素材整理为公众号长文。覆盖从意图理解、大纲设计、初稿撰写到审读自检的完整写作工作流。

## 功能特性

- **9 步写作工作流**：从理解意图到终稿完成，每一步有明确的输入输出和质量标准
- **独立审读 + 事实核查**：通过 subagent 对初稿进行 AI 味检测、逻辑连贯性检查、事实性核查，避免 AI 生成内容的常见问题
- **活人感终审**：以读者视角判断文章读起来是「朋友在聊天」还是「AI 在输出」
- **风格校准**：通过范文分析 + 行文风格指南双重校准，确保输出文章的语感与作者一致
- **自检清单**：硬性规则、风格一致性、HKR 质检、活人感四层自检，附带结构化报告

## 快速开始

### 安装

将本项目克隆到 Claude Code 的 skills 目录下：

```bash
git clone https://github.com/<your-username>/mp-article-writor.git ~/.claude/skills/mp-article-writor
```

或者将项目文件放置到你的 Claude Code skills 路径中。

### 自定义配置

安装后，你需要根据自身情况修改 `SKILL.md` 中的以下内容：

- **公众号名称**：全文搜索替换为你自己的公众号名
- **固定结尾**：替换文末的推广段落为你自己的内容
- **作者声音**：修改「作者声音」章节，描述你自己的写作调性
- **读者画像**：修改「读者画像」章节，定义你的目标读者
- **文件保存路径**：修改文档保存路径和 front matter tags
- **范文风格分析**：将 `references/范文风格分析.md` 替换为你自己历史文章的风格提炼（参见该文件开头的方法论说明）

### 使用

在 Claude Code 中直接描述你的写作需求即可触发 Skill，例如：

- 「帮我写一篇关于 XX 的公众号文章」
- 「把这些素材整理成推文」
- 「发公众号」

Skill 会按 9 步工作流依次执行，每步完成后再进入下一步。

## 工作流概览

```
Step 1  理解作者意图    → 确认切入角度、深度、主旨、素材
Step 2  阅读参考资料    → 校准语感（范文分析 + 风格指南）
Step 3  设计大纲和风格  → 呈现大纲，等待确认
Step 4  编写初稿        → 按规则写作
Step 5  独立审读        → AI 味检测、逻辑、结构、信息密度
Step 6  事实核查        → 数据来源、场景真实性、术语拼写
Step 7  修改初稿        → 逐条处理反馈
Step 8  终审自检        → 四层自检清单
Step 9  完成终稿        → 更新文件 + 自检报告 + 标题推荐
```

## 自定义范文风格分析

`references/范文风格分析.md` 是 Skill 校准语感的依据。你需要用自己的历史文章替换其中的示例分析。

提炼方法：从你自己的公众号文章中选取 2-3 篇风格最满意的，按以下维度逐篇拆解：

1. **叙事策略**：文章用了什么叙事框架？（英雄之旅 / 悬疑破案 / 横向对比 / 对话体……）
2. **结构拆解**：把文章切成段落级的功能块，标注每段的叙事角色（钩子 / 方法论 / 转折 / 结论 / 收束）
3. **关键技法**：哪些写作手法是这篇文章的亮点？具体到句子级别举例
4. **语言特征**：用 3-5 句话概括这篇文章的语感（正式/随意、冷静/热情、技术/生活化……）

拆解完成后，在末尾写一段「跨文章共性模式」，总结所有范文中反复出现的风格特征。这些共性模式就是 Skill 生成文章时的语感锚点。
