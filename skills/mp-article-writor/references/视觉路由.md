# 微信公众号视觉路由

本文用于规划和生产微信公众号「高效人生指北」文章的静态封面、正文解释性插图和真实证据素材。开始 Step 3 前阅读，执行 Step 10 和 Step 11 时再次核对。

## 固定边界

- 目标平台只允许微信公众号。
- 交付形式只允许静态图文。
- 禁止生成 Live Photo、MOV、PVT、GIF、MP4、视频替代品、小红书图片和其他平台封面。
- 公众号封面固定使用 `guizang-social-card-skill`。
- 正文解释性插图在 Step 1 由作者选择 `guizang-social-card-skill` 或 `guizang-material-illustration`。
- 同一篇文章默认只使用一种归藏正文插图风格。作者明确要求混用时，才为具体插图分别指定 Skill。
- 真实截图、照片和原始图表可以与任一风格共存，它们属于证据素材。

## 安装与可用性检查

完整静态视觉工作流推荐安装以下两个 Skill：

```bash
npx skills add https://github.com/op7418/guizang-social-card-skill --skill guizang-social-card-skill
npx skills add https://github.com/op7418/guizang-material-illustration --skill guizang-material-illustration
```

在 Step 1 开始时检查两个 Skill 是否可读取：

- 两个都已安装：正常询问正文解释性插图风格并执行完整视觉生产。
- 只安装 social-card：可以生产公众号双封面和 social-card 正文插图；material-illustration 选项需要安装后才能执行。
- 只安装 material-illustration：可以生产 material 正文插图；公众号双封面需要安装 social-card 后才能执行。
- 两个都未安装：继续完成 Step 1 至 Step 9 的文章工作，Step 10 不生产归藏视觉素材，Step 11 列出安装命令和待完成项目。

安装提示只显示一次。不得自动安装，不得阻塞文章写作，不得生成旧版 4:3 题图或正文插图 prompt 作为替代，也不得把用户选择静默改成另一个 Skill。

## 首次询问

向作者说明两种正文解释性插图风格的实际差异，并给出结合文章内容的推荐：

`guizang-social-card-skill`

- 生成带完整文字层级和版式关系的静态卡片。
- 可选杂志编辑风或瑞士国际主义风格。
- 适合观点摘要、步骤说明、截图讲解、前后对比、数据摘要和需要较多排版文字的解释图。
- 交付静态 PNG 和可编辑 HTML。

`guizang-material-illustration`

- 生成 3D 瑞士编辑风格的材质插图。
- 适合循环、流程、系统关系、层级、机制、场景化概念和数据优先的编辑型图表。
- 图内只保留 2 至 5 字的短标签，完整解释放在文章正文。
- 交付静态栅格图片和 `PROMPTS.md`。

询问只执行一次。作者选择后，将选择结果写入视觉脚本。不得在生产阶段再次询问同一选择。

## 视觉素材清单

在大纲阶段为每项素材记录：

```text
素材编号：V01
所在章节：
视觉作用：证据 / 解释 / 氛围
支撑内容：对应文章中的具体事实、观点或步骤
素材类型：截图 / 照片 / 原始图表 / social-card / material-illustration
来源：作者提供 / 本地文件 / 公开来源 / AI 生成
本地路径：
授权状态：作者自有 / 已授权 / 需署名 / 待确认
引用要求：
生成 Skill：guizang-social-card-skill / guizang-material-illustration / 无
目标比例和尺寸：
新增中文标签：
裁切重点：
移动端检查重点：
最终状态：待准备 / 待生成 / 已通过 / 需替换
```

视觉作用必须清楚。无法支撑文章内容、只用于填空的图片不进入清单。

## 路由规则

真实证据素材优先

- 产品体验、教程、测评和个人实践优先使用作者的真实截图、照片、数据和结果。
- 简单证据图直接插入正文。
- 需要重点标注、局部放大、并列对比或文字解释时，交给作者选定的正文插图 Skill。
- 不得重绘截图中的产品界面并将其当作真实证据。

公众号封面

- 使用 `guizang-social-card-skill`。
- 主封面为 `21:9`，输出 `2100×900`。
- 分享封面为 `1:1`，输出 `1080×1080`。
- 两个比例分别构图并放入同一 HTML 预览。1:1 使用提炼后的短标题，不得直接裁切主封面。
- 调用中显式禁止 Live Photo 和其他平台输出。

正文使用 social-card

- 适合文字与视觉关系同等重要的解释图。
- 从 Editorial Magazine x E-ink 或 Swiss International 中选择一种模式，并让整组正文插图保持一致。
- 使用公众号正文适合的静态比例；根据内容选择 `1.9:1`、`16:9` 或保留截图原始比例，不强制 `4:3`。
- 每张图输出 PNG；涉及排版的图片保留可编辑 HTML。

正文使用 material-illustration

- 适合把一个概念压缩成清晰的物体关系、流程、循环、层级或数据场景。
- 整组使用同一种强调色和统一的 3D 瑞士编辑视觉系统。
- 默认优先使用 `1.9:1`；标签或关系较多时使用 `16:9`；只有明确需要方形插图时使用 `1:1`。
- 图内标签保持 2 至 5 个汉字，每张图通常使用 3 至 5 个标签。
- 图表必须逐项传递准确数值、分类顺序、坐标轴、单位和误差范围。任何数字错误都必须重新生成。

## 输出目录

每篇文章使用独立目录：

```text
projects/自媒体运营/mp-高效人生指北/article-assets/<文章标识>/
├── cover/
│   ├── wechat-main-21x9.png
│   ├── wechat-share-1x1.png
│   └── index.html
├── illustrations/
│   ├── V01-<简短名称>.png
│   └── ...
├── evidence/
│   ├── V02-<简短名称>.<扩展名>
│   └── ...
├── PROMPTS.md
└── SOURCES.md
```

所有归藏 Skill 都必须接收该明确目录。归藏仓库内的 `local-tests` 只可用于临时调试，不能作为最终交付位置。

`PROMPTS.md` 记录使用的 Skill、视觉模式、配色、最终 prompt、输出路径和被拒绝版本的原因。`SOURCES.md` 记录本地原始路径、公开来源链接、授权状态、署名要求和仍待确认的问题。

## 授权和自动检查

- Step 1 对所选归藏 Skill 的首次渲染确认，同时视为授权执行后续静态检查、修复和重新渲染。
- 不在首次渲染后询问是否自动检查。
- 自动检查不扩大内容范围，不新增未确认的外部素材，不改变作者选定的正文插图风格。
- 外部素材自动记录来源和授权状态。状态无法确认时标记「待作者确认」，不得写成已授权。

## 交付检查

- 两个公众号封面尺寸正确并分别构图。
- 正文解释性插图使用作者选定的 Skill。
- 图片内容支撑视觉脚本中指定的事实、观点或步骤。
- 所有中文标签清楚、正确、没有截断或乱码。
- 真实截图和照片没有拉伸，也没有裁掉关键界面、人物或产品细节。
- 图表数字、顺序、坐标轴、单位和正文完全一致。
- 所有最终文件位于明确输出目录，文章中的引用路径可以读取。
- `PROMPTS.md` 和 `SOURCES.md` 完整。
- 两个归藏 Skill 的可用状态已记录，缺失依赖对应的视觉项目已列入待完成清单。
- 不存在 Live Photo、MOV、PVT、GIF、MP4、视频或其他平台输出。
